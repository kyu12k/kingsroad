## 1. 게임 소개

**King's Road**는 요한계시록(신약성경 마지막 책, 22장)을 암송하기 위한 **게임형 훈련 앱**입니다. 스마트폰이나 PC 브라우저에서 바로 실행되며, 앱으로 설치해 오프라인에서도 사용할 수 있습니다.

**대상 사용자**: 요한계시록 암송을 목표로 하는 한국어 사용 기독교인, 교회 청년부, 성경 암송 모임 등

**핵심 목표**: 암송이 지루하거나 부담스럽지 않도록, 게임의 재미 요소(보상, 경쟁, 성장)를 활용해 꾸준한 훈련을 돕는 것

---

## 2. 핵심 기획 콘셉트

이 게임의 설계 핵심은 두 가지 원리의 결합입니다.

### 망각 곡선 (Forgetting Curve)
인간은 배운 내용을 시간이 지나면 잊습니다. 이를 극복하려면 **적절한 시점에 반복 복습**해야 합니다. King's Road는 에빙하우스의 망각 곡선 이론을 기반으로, 처음 암송한 구절을 **1일 → 3일 → 7일 → 14일 → 30일** 간격으로 복습하도록 유도합니다.

### 게임화 (Gamification)
단순 반복 암송이 되지 않도록 다음 요소들을 통해 동기를 유지합니다:
- **보상**: 훈련 완료 시 보석 획득
- **성장**: 성전 업그레이드, 레벨 상승
- **경쟁**: 전국 랭킹, 리그 시스템
- **도전**: 보스 배틀, 심화 훈련, 업적
- **습관 형성**: 일일/주간 미션, 출석 체크

## 3. 콘텐츠 구조

```
요한계시록 22장
└── 각 장(Chapter) 1~22
    ├── 스테이지 (3~7개 구절 단위로 구성)
    ├── 중간점검 보스 (체크포인트, 구절 묶음마다)
    └── 파이널 보스 (각 장 마지막)
```

- 전체 구절을 **3~7절 단위 스테이지**로 나눠 학습 부담을 줄였습니다.
- 스테이지를 완료하면 **중간점검(체크포인트) 보스**가 등장합니다.
- 한 장 전체를 완료하면 **파이널 보스**를 처치해야 다음 장이 열립니다.
- 체크포인트를 통과하면 그 범위 내 **미학습 스테이지가 복습 모드로 즉시 개방**됩니다.

## 4. 훈련 시스템 — 5단계 훈련

각 스테이지는 **5단계 순서로 진행**됩니다. 모든 단계를 완료해야 그 스테이지의 초기 학습이 끝납니다.

| 단계 | 이름 | 내용 |
|------|------|------|
| Step 1 | 읽기 | 구절 전체를 눈으로 읽으며 익힘 |
| Step 2 | 초성 암기 | 초성을 보고 알맞은 단어 넣기 |
| Step 3 | 객관식 암기 | 세 개의 단어 중 알맞은 단어를 선택 |
| Step 4 | 시간제한 암기 | 흘러가는 구절 속 빈칸에 들어갈 단어 넣기 |
| Step 5 | 완전 암송 | 주어진 단어를 올바른 순서로 배치 |

**학습 상태 구분**:
- **전체학습 모드**: Step 1~5를 아직 완료하지 않은 스테이지에 진입 시 자동 적용
- **복습 모드**: Step 1~5를 모두 완료한 스테이지에 재진입 시 적용 (Quick: [2,5단계] 또는 Full: [1~5단계])

시간 간격 기반 복습(1일·3일·7일 등)은 아래 **망각 곡선 시스템**에서 담당합니다.

**전체학습(Full-New) 모드**: 체크포인트 통과 후 개방. 이미 학습한 여러 구절을 한 번에 5단계 훈련.

## 5. 심화 훈련 3종

일반 훈련보다 높은 난이도의 심화 훈련입니다. 장별로 진입할 수 있습니다.

| 모드 | 이름 | 방식 |
|------|------|------|
| 인내 | 인내 훈련 | 구절을 읽으며 정확도를 유지하는 연속 훈련. 틀리면 HP 감소 |
| 기억 | 기억 훈련 | 구절 전체를 타이핑으로 입력. 시각적 힌트 없음 |
| 주소 | 주소 훈련 | 구절 내용을 보고 장·절 번호(주소)를 맞히는 훈련 |

## 6. 망각 곡선 & 복습 시스템

Step 1~5 완료 이후에도 망각 방지를 위한 장기 복습이 진행됩니다.

### 복습 주기

| 기억 레벨 | 복습 가능 시점 | 설명 |
|-----------|--------------|------|
| 레벨 0 | 23시간 후 | 첫 학습 직후 |
| 레벨 1 | 71시간 후 | 1차 복습 완료 |
| 레벨 2 | 167시간 후 | 2차 복습 완료 |
| 레벨 3 | 335시간 후 | 3차 복습 완료 |
| 레벨 4+ | 719시간 후 | 장기 기억 단계 |

### 양식 보너스 (잊어버린 구절 복습 시)

추천 복습 대기 시점이 지난 구절을 복습하면 추가 보석 보너스가 지급됩니다:

| 복습 순서 | 보너스 배율 | 대기 시간 |
|-----------|------------|----------|
| 1회 | 1.5배 | 10분 후 |
| 2회 | 2배 | 1시간 후 |
| 3회 | 5배 | 6시간 후 |

## 7. 보스 배틀

스테이지 묶음의 마지막마다 보스 배틀이 등장합니다.

- **중간점검 보스**: 3~7절 묶음 완료 후 등장
- **파이널 보스**: 각 장(Chapter) 완료 후 등장
- **HP 시스템**: 보스의 HP = 해당 범위 구절 수. 틀린 답변 시마다 HP 1 감소
- **패배 시**: 보석을 소모해 부활하거나, 포기하면 홈으로 이동
- **승리 시**: 다음 스테이지 개방 + 보석 보상
- **아이템**: 생명의 떡(HP 회복 아이템)을 사용해 배틀 중 HP 복구 가능

## 8. 성장 시스템 — 성전 & 자원

### 자원

| 자원 | 설명 |
|------|------|
| 보석 | 주 화폐. 훈련 완료·미션·오프라인 생산으로 획득. 성전 업그레이드에 사용 |
| 생명의 떡 | 보스 배틀 중 HP 회복 소모 아이템 |

**보석 획득량 예시**:
- 일반 스테이지 클리어: 5~70보석 (모드·난이도에 따라 다름)
- 망각 복습 보너스: 최대 5배
- 성전 보너스 추가: +2%~+30%

### 성전 (Castle) 업그레이드

총 12레벨. 성전 레벨이 오를수록 아래 혜택이 증가합니다:

| 혜택 | 내용 |
|------|------|
| 오프라인 생산 | 접속하지 않아도 시간당 보석 자동 생산 (5~150개/시간) |
| 저장 한도 | 최대 보석 보유량 증가 (40~1,500개) |
| 보너스 배율 | 스테이지 클리어 시 보석 보너스 +2%~+30% |

성전 테마는 성경적 이름으로 구성됩니다 (Lv.1 "제사장의 시작" ~ Lv.12 "열린 문과 영광").

## 9. 경쟁 시스템 — 랭킹 & 리그

### 랭킹 구조

| 탭 | 설명 |
|------|------|
| 시온성 랭킹 | 주간 점수 기준 상위 100명. 매주 월요일 초기화 |
| 주간 명예의 전당 | 지난 주 상위 기록 보관 |
| 월간 명예의 전당 | 지난 달 상위 기록 보관 |
| 누적 명예의 전당 | 전체 기간 누적 점수 기준 기록 |

### 점수 종류

- **주간 점수**: 매주 월요일 초기화
- **월간 점수**: 매월 1일 초기화
- **누적 점수**: 삭제되지 않는 전체 누적 점수
- **연간 점수**: 연도별 점수

### 명예의 전당

주간/월간 랭킹 마감 시 상위 기록이 자동 보관됩니다. 과거 기록을 열람할 수 있습니다.

### 자동 관리 (Firebase Cloud Functions)

| 기능 | 설명 |
|------|------|
| `updateZionCutoff` | 시온 100명 제한 자동 관리 |
| `archiveMonthlyRankings` | 월간 기록 자동 백업 |
| `checkZionEligibility` | 클라이언트가 시온 자격 여부 확인 |

## 10. 미션 & 업적

### 일일 미션 (매일 초기화, 3종)

| 미션 | 조건 |
|------|------|
| 신규 클리어 | 새 스테이지 1회 이상 클리어 |
| 다양한 훈련 | 서로 다른 스테이지 훈련 |

| 미션 | 조건 |
|------|------|
| 출석 | 주 7일 접속 |
| 용 처치 | 주간 보스 처치 |
| 스테이지 클리어 | 주간 스테이지 다수 클리어 |

### 업적 시스템 (8단계 티어)

다양한 카테고리에서 누적 달성 시 보석 보상:
- 로그인 일수 / 클리어 구절 수 / 보스 처치 수 / 획득 보석 / 완벽 점수 / 성전 레벨업 / 얼리버드 보너스 등

## 11. 기술 스택

| 항목 | 기술 |
|------|------|
| 프론트엔드 | HTML5, CSS3, Vanilla JavaScript |
| 오디오 | 효과음 시스템 (Web Audio API) |
| 백엔드 | Firebase (Firestore DB, Cloud Functions) |
| 인증 | Firebase Authentication |
| 오프라인 | Service Worker (PWA), LocalStorage |
| 배포 | Firebase Hosting |

**데이터 저장 방식**:
- **LocalStorage**: 모든 게임 진행상황 로컬 저장 (오프라인 우선)
- **Firebase Firestore**: 랭킹 동기화, 명예의 전당 기록

**PWA**: 스마트폰 홈 화면에 앱으로 설치 가능, 오프라인 플레이 지원

## 12. 파일 구조

```
kingsload/
├── index.html                  # 전체 UI 구조 및 모달 (89KB)
├── game.js                     # 전체 게임 로직 (721KB, 13,758줄)
├── style.css                   # 전체 스타일 (162KB, 4,966줄)
├── sw.js                       # 서비스 워커 (오프라인 PWA)
├── firebase-messaging-sw.js    # Firebase 메시징 서비스 워커
├── manifest.json               # PWA 앱 매니페스트
├── firebase.json               # Firebase 호스팅 설정
├── firestore.rules             # Firestore 보안 규칙
├── firestore.indexes.json      # Firestore 인덱스 정의
├── icon.png                    # 앱 아이콘 (512x512)
├── icon-192.png                # 앱 아이콘 (192x192)
├── CHANGES_FULL_LEARNING.md    # 전체학습 해제 기능 변경 이력
├── DEPLOYMENT_GUIDE.md         # Cloud Functions 배포 가이드
└── functions/
    ├── index.js                # Cloud Functions 3개 (랭킹 관리)
    └── package.json            # Node.js 의존성
```

**주요 파일 설명**:

- **game.js**: 게임의 모든 로직이 이 파일 하나에 담겨 있습니다. 307개 함수가 포함되며, 훈련 시스템·보스 배틀·랭킹·미션·업적·UI·오디오 등 전체를 담당합니다.
- **index.html**: 화면 구조와 모든 모달(팝업)의 HTML이 담겨 있습니다. JS에서 이 모달들을 show/hide 방식으로 제어합니다.
- **functions/index.js**: Firebase Cloud Functions. 랭킹 자동 관리(시온 100인 제한, 월간 기록 백업)를 담당합니다.

## 13. 배포 & 환경 설정

자세한 내용은 [DEPLOYMENT_GUIDE.md](./DEPLOYMENT_GUIDE.md)를 참고하세요.

**필요 조건**:
- Firebase 프로젝트 (Blaze 플랜 — Cloud Functions 사용을 위해 필요, 실제 비용은 거의 없음)
- Node.js (Cloud Functions 배포 시)
- Firebase CLI

**빠른 시작**:
```bash
# Firebase CLI 설치
npm install -g firebase-tools

# 로그인
firebase login

# 배포
firebase deploy
```
